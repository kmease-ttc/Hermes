## Part A — Define 3 states for every dashboard card

Every card (metrics, modules, analytics, technical, etc.) must render one of:

### 1) ACTIVE (has data)

* Show real number/value
* Normal styling

### 2) SETUP REQUIRED (feature exists but not connected/configured)

Examples: GA4 not connected, GSC not connected, domain not verified, no keywords selected.

UI rules:

* Show value as “—”
* Show badge: “Requires setup”
* Primary CTA inside card: “Connect” or “Finish setup”
* Entire card is clickable → routes to the exact config page
* Avoid low-contrast gray-on-gray. Use white background + border + small muted text.

### 3) LOCKED / NOT PURCHASED (needs subscription/module)

UI rules:

* Show value as “—”
* Show badge: “Locked”
* CTA inside card: “Subscribe”
* Entire card is clickable → routes to Agents/Plans section (or purchase modal/page)
