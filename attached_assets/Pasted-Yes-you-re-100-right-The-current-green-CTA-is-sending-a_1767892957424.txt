Yes — you’re 100% right. The current green CTA is sending a mixed brand signal because the logo is **not** green-forward. The easiest way to make the whole site feel “designed” is:

1. **Derive the CTA colors directly from the logo**, and
2. Keep green only as a subtle “success” state (optional), not the primary brand action.

From your logo, the brand reads as a **purple → pink → amber** gradient (the “A” mark). So your buttons should live in that world.

## What to change

### 1) Replace the primary CTA green with a logo-gradient button

Use a gradient pulled from the logo family:

**Primary CTA gradient**

* Start: `#8B5CF6` (violet)
* Mid: `#EC4899` (pink)
* End: `#F59E0B` (amber)

That will instantly harmonize with the logo and remove the “random green” feel.

### 2) Keep a single accent system across the page

* Links, highlights, icon badges, and CTA should all use the same gradient family.
* If you need green at all, use it only for semantic success states (checkmarks), not the brand CTA.

## Implementation (Tailwind-ready)

### Button classes

Use this for your primary buttons (“Fix it”, “Fix my SEO automatically”, etc.):

```tsx
className="
  inline-flex items-center justify-center rounded-xl px-5 py-3 text-sm font-semibold text-white
  bg-gradient-to-r from-violet-500 via-pink-500 to-amber-400
  shadow-[0_14px_30px_rgba(139,92,246,0.20)]
  hover:shadow-[0_18px_40px_rgba(236,72,153,0.22)]
  hover:-translate-y-0.5 transition
  focus:outline-none focus:ring-4 focus:ring-violet-200
"
```

### Secondary button (outline, still on-brand)

```tsx
className="
  inline-flex items-center justify-center rounded-xl px-5 py-3 text-sm font-semibold
  text-slate-900 bg-white border border-slate-200
  hover:border-slate-300 hover:bg-slate-50 transition
"
```

### Links (logo-aligned)

```tsx
className="text-violet-600 hover:text-pink-600 font-medium"
```
