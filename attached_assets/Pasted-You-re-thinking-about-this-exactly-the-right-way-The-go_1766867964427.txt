1) The mental model (this matters more than the colors)

You need three layers of color, not one:

Layer 1: Structural colors (never change)

These define surfaces, borders, text.
They make the app feel cohesive.

Layer 2: Semantic colors (meaningful)

These answer: what kind of thing is this?
Examples:
	•	performance
	•	risk
	•	opportunity
	•	progress
	•	blocked

Layer 3: Accent colors (action & delight)

These guide attention:
	•	CTAs
	•	selections
	•	unlocks
	•	progress moments

If you mix these layers, things look noisy.
If you separate them, everything clicks.

2) Base dark palette (structural)

These are your non-negotiables.

Backgrounds

bg-app        #070A14  (deep space navy)
bg-surface-1  rgba(255,255,255,0.04)
bg-surface-2  rgba(255,255,255,0.07)
bg-surface-3  rgba(255,255,255,0.10)

Borders & dividers

border-subtle rgba(255,255,255,0.10)
border-active rgba(255,255,255,0.20)

Text

text-primary   rgba(255,255,255,0.92)
text-secondary rgba(255,255,255,0.72)
text-muted     rgba(255,255,255,0.48)

Everything else sits on top of this.

4) Semantic color set (THIS solves your bullet problem)

This is what you’re missing right now.

Instead of coloring bullets arbitrarily, every bullet should map to one semantic intent.

Semantic colors (muted, dark-mode friendly)

semantic-success  #22C55E  (green)
semantic-warning  #FBBF24  (amber)
semantic-danger   #EF4444  (red)
semantic-info     #3B82F6  (blue)
semantic-neutral  rgba(255,255,255,0.55)

Important:
These should almost never be full backgrounds.
They should appear as:
	•	icons
	•	bullets
	•	left borders
	•	small pills
	•	text accents

5) How to apply this to your UI (concrete examples)

A) Bullet points in cards (your main pain point)

Instead of random colors, do this:

Example: Visibility Intelligence card
	•	Keyword tracking → info (blue)
	•	Rank change explanations → info (blue)
	•	Technical SEO scoring accuracy → progress (gold)
	•	Conversion-weighted fixes → success (green)

Visually:
	•	bullet dot or icon is colored
	•	text stays text-secondary

This gives variation without chaos.

⸻

B) Primary actions (buttons)

Primary CTA:
- bg: accent-primary
- text: white
- hover: slightly brighter purple

Secondary CTA:
- bg: bg-surface-2
- border: accent-primary-soft
- text: accent-primary

Never color buttons green/red/yellow unless they’re destructive.

⸻

C) Status badges (Good / Watch / Alert)

Badges only. Never whole cards.

Good    → green pill, low opacity
Watch   → amber pill
Alert   → red pill

Background: rgba(color, 0.15)
Border: rgba(color, 0.35)

⸻

D) Set Bonuses cards
	•	Title: text-primary
	•	Value prop: text-secondary
	•	Progress number: gold
	•	“Next to unlock” CTA: purple
	•	Missing chips: semantic-neutral
	•	Completed requirements: green check

⸻

E) Crew tiles
	•	Active → gold ring
	•	Selected → white ring
	•	Empty → dashed neutral border
	•	Errors / missing config → red badge only

No backgrounds colored.

⸻

6) Guardrails (so it doesn’t spiral)

These rules will save you later:
	1.	Never use more than 2 accent colors in one card
	2.	Never color the entire background of a card for status
	3.	Text is almost always white/gray — color belongs to icons & accents
	4.	Purple = action, Gold = progress, Semantic colors = meaning
	5.	If you can’t explain why a color is used, it’s wrong

7) Replit / ArcGlow prompt to implement this cleanly

You can paste this directly:

DARK THEME COLOR SYSTEM EXPANSION (NON-MONOTONE)

Goal:
Implement a dark UI color system that supports dashboards, actions, bullets, and status without becoming monochromatic or noisy.

Requirements:
1) Define and use three color layers:
- Structural (backgrounds, borders, text)
- Accent (primary action, progress/unlocks)
- Semantic (success, warning, danger, info)

2) Structural colors:
- Deep navy app background
- Dark translucent surfaces for cards
- Subtle borders
- High-contrast white/gray text

3) Accent usage:
- Purple ONLY for primary actions, links, selections
- Gold ONLY for progress, unlocks, maturity, active crew
- Cyan optional for system/engine indicators

4) Semantic colors:
- Green: success/outcomes
- Amber: warnings/watch states
- Red: errors/blockers
- Blue: informational signals
- Use these ONLY for icons, bullets, badges, small accents
- NEVER use them as full card backgrounds

5) Bullet points:
- Each bullet must map to a semantic intent
- Colored icon/bullet, neutral text
- No random or decorative color usage

6) Apply consistently across:
- KPI cards
- Agent summary cards
- Set bonus cards
- Capability cards
- Blockers panel
- Buttons and links

Acceptance criteria:
- UI is visually rich but controlled
- Dark theme feels intentional, not flat or neon
- Bullets and badges communicate meaning clearly
- Purple + gold are accents, not crutches
