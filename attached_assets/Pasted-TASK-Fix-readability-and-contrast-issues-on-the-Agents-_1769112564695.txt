TASK: Fix readability and contrast issues on the Agents page (`/app/agents`)

GOAL
Make the Agents page fully readable. Increase contrast for:

* Left-hand navigation
* Top-of-page content (headers, controls, metadata)
* Agent cards (titles, descriptions, status text)

This is a contrast + hierarchy fix, NOT a redesign.

---

SCOPE
Files likely involved:

* `src/components/Sidebar.tsx` or `LeftNav.tsx`
* `src/pages/agents.tsx` (or equivalent)
* `src/components/AgentCard.tsx`
* `src/styles/*` or `index.css`
* Tailwind config if tokens are defined there

---

1. LEFT-HAND NAV (PRIMARY ISSUE)

Problem:
Left nav background and text are too light, causing poor contrast and weak visual anchoring.

Fix:

* Darken the nav background
* Increase text contrast
* Ensure active + hover states are clearly visible

Implementation:

* Set nav background to a dark solid color (not translucent)
  Example:

  * Background: `bg-slate-900` or `bg-purple-900`
* Set nav text to `text-slate-100`
* Inactive items: `text-slate-300`
* Hover: `hover:text-white hover:bg-slate-800`
* Active item:

  * `bg-slate-800`
  * `text-white`
  * Left border or indicator (2–3px)

Do NOT use:

* `opacity-*` on the nav container
* translucent backgrounds like `bg-*/40`

---

2. TOP-OF-PAGE CONTENT (SECONDARY ISSUE)

Problem:
Page title, subtitles, filters, and controls at the top are washed out and hard to scan.

Fix:

* Explicitly set text colors
* Increase font weight for hierarchy
* Remove inherited opacity

Implementation:

* Page title: `text-slate-900 font-semibold`
* Section headers: `text-slate-800 font-medium`
* Meta / helper text: `text-slate-600`
* Buttons and toggles:

  * Ensure labels are `text-slate-700` or darker
  * Icons should NOT rely on opacity alone

Audit:

* Inspect computed styles and confirm `opacity: 1` on text elements

---

3. AGENT CARDS (CONFIRM + ENFORCE)

Problem:
Agent cards still inherit low contrast from parent styles.

Fix:

* Ensure cards use solid backgrounds
* Never reduce opacity on the card container
* Mute only non-text elements (icons, badges)

Implementation:
Card container:

* `bg-white`
* `border border-slate-200`
* `text-slate-900`

Text inside cards:

* Agent name: `text-slate-900 font-medium`
* Description: `text-slate-700`
* Status / metadata: `text-slate-600`

Inactive state:

* Change background slightly (`bg-slate-50`)
* DO NOT change text opacity
* If needed, dim icons only

Explicitly remove:

* `opacity-*`
* `text-opacity-*`
* `bg-white/50`

---

4. GLOBAL SAFETY NET (IF NEEDED)

If contrast keeps regressing due to inherited styles, add a scoped override:

CSS:

```
.agent-card,
.agent-card * {
  opacity: 1 !important;
}
```

Apply `agent-card` class to the Agent card root.

This is acceptable as a temporary guardrail.

---

5. VALIDATION CHECKLIST (REQUIRED)

Before marking done:

* Can read agent names without squinting
* Can read descriptions without zoom
* Left nav text is readable at a glance
* Active nav item is obvious
* No text relies on opacity for hierarchy

Use DevTools:

* Click text → computed styles
* Confirm `opacity: 1`
* Confirm text colors are ≥ slate-600 equivalent

---

DONE WHEN:
The page is readable on a normal laptop at 100% zoom with no visual strain.
No transparency is used to “mute” text anywhere on `/app/agents`.
