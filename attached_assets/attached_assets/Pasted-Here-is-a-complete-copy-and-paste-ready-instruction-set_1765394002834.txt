1. ON-PAGE SEO STRUCTURE (MANDATORY IMPLEMENTATION)

1.1 Title Tags

Implement dynamic SEO title fields for each page.
Rules:
	•	≤ 60 characters
	•	Format: Primary Keyword | Arclo
	•	Auto-title fallback if field is empty
	•	Use Title Case

Developer Action:
Add editable title field to CMS; inject into <title>.

⸻

1.2 Meta Descriptions

Rules:
	•	120–160 chars
	•	Keyword + clear value proposition
	•	Include CTA (“Generate your site now” etc.)

Developer Action:
Add editable meta description field to CMS; auto-generate fallback using text summarizer API (OpenAI or similar).

⸻

1.3 H1 / H2 / H3 Structure
	•	Exactly one <h1> per page.
	•	Use keyword-driven H2s in content sections.
	•	Auto-lint pages to prevent multiple H1s.

Developer Action:
Add a simple validation rule in CMS editor or at build time.

⸻

1.4 URL Slug Optimization

Rules:
	•	Lowercase
	•	Hyphens only
	•	No dynamic parameters
	•	No stopwords

Developer Action:
Add slug normalization middleware.

⸻

2. CONTENT OPTIMIZATION UPDATES

2.1 Thin Content Prevention

Every page must have ≥ 600 words of crawlable text.

Developer Action:
Add required “SEO content section” to each major template (homepage, feature pages, pricing page, blog pages).
Styling should hide overflow until expanded (UX not hurt).

⸻

2.2 FAQ Blocks

Every page gets an optional FAQ block.

Developer Action:
Add component:
	•	Question
	•	Answer
	•	Auto-generates FAQ Schema

Arclo’s marketing team will populate.

⸻

2.3 Blog System

Use a headless blog service rather than building your own.

RECOMMEND:
	•	Ghost (SEO-optimized; API; fast)
or
	•	Hashnode Pro (programmatic publishing)

Developer Action:
	•	Integrate blog subdomain: arclo.pro/blog
	•	Use reverse proxy or direct link
	•	Add sitemap injection to main sitemap.xml

3. TECHNICAL SEO INFRASTRUCTURE

3.1 XML Sitemaps

Implement:
	•	/sitemap.xml (master index)
	•	/sitemap-pages.xml
	•	/sitemap-blog.xml
	•	/sitemap-images.xml

Developer Action:
Use a 3rd-party auto-sitemap generator:
	•	Yoast Headless Sitemap API or
	•	SimpleSitemap.com (API-based)

Avoid building your own.

⸻

3.2 robots.txt

Required entries:

User-agent: *
Allow: /
Sitemap: https://arclo.pro/sitemap.xml


⸻

3.3 Core Web Vitals

Implement:
	•	Lazy loading images
	•	Script defer/async
	•	Preload hero image
	•	Preload main font

Developer Action:
Use NitroPack, Cloudflare APO, or Vercel Performance depending on your hosting.

Avoid hand-optimizing assets.

⸻

4. SCHEMA / STRUCTURED DATA

Mandatory Schemas:
	•	Organization
	•	WebSite
	•	FAQPage (if FAQ block exists)
	•	SoftwareApplication (SaaS product)

Developer Action:
Create a schema generator module that:
	•	Accepts config object
	•	Outputs JSON-LD into <script type="application/ld+json">

Do not hand-write JSON every time; use a reusable template.

⸻

5. INTERNAL LINKING SYSTEM

Requirements:
	•	Auto-generate navigation breadcrumbs
	•	Footer quick links
	•	Blog → product page cross-links
	•	“Related content” suggestions on blog posts

Developer Action:
Use simple deterministic rules:
	•	If tag = “automation”, relate all automation pages
	•	If tag = “seo”, relate all SEO pages
	•	Blog → main features page link added automatically in footer of post template

No need for machine learning here.
