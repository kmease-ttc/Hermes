Goal: replace the default Replit favicon with the Arclo favicon across all browsers, tabs, and devices.

1. Create proper favicon assets

* Generate the following files from the Arclo logo (no accent over the “o”):
  * favicon.ico (16x16, 32x32, multi-size)
  * favicon-16x16.png
  * favicon-32x32.png
  * apple-touch-icon.png (180x180)
  * android-chrome-192x192.png
  * android-chrome-512x512.png
* Background should be transparent or match the site background.
* Name files exactly as above (no spaces, lowercase).

2. Place favicon files in the correct public directory

* Put all favicon files in the site’s public/static root so they resolve at:
  * https://arclo.pro/favicon.ico
  * https://arclo.pro/favicon-32x32.png
  * etc.
* Do NOT rely on framework defaults or Replit-generated assets.

3. Explicitly declare favicons in the HTML <head>
   Add all of the following to the base HTML template (index.html, _document.tsx, or equivalent):

<link rel="icon" href="/favicon.ico" sizes="any">
<link rel="icon" type="image/png" sizes="32x32" href="/favicon-32x32.png">
<link rel="icon" type="image/png" sizes="16x16" href="/favicon-16x16.png">
<link rel="apple-touch-icon" href="/apple-touch-icon.png">
<link rel="manifest" href="/site.webmanifest">

Important:

* This must live in the global document/head, not per-page components.
* Do not depend on framework auto-injection.

4. Remove or override Replit defaults

* Search the codebase for any existing favicon references (Replit defaults often include their own favicon).
* Remove any:
  * <link rel="icon" href="replit…">
  * Auto-generated meta/favicon config from Replit templates.
* Ensure only Arclo favicon links remain.

5. (Optional but recommended) Add site.webmanifest
   Create /site.webmanifest with:
   {
   "name": "Arclo",
   "short_name": "Arclo",
   "icons": [
   { "src": "/android-chrome-192x192.png", "sizes": "192x192", "type": "image/png" },
   { "src": "/android-chrome-512x512.png", "sizes": "512x512", "type": "image/png" }
   ],
   "theme_color": "#000000",
   "background_color": "#000000",
   "display": "standalone"
   }

6. Cache busting + verification

* After deploy:
  * Hard refresh (Cmd+Shift+R)
  * Open site in an incognito window
  * Check tab icon, bookmarks icon, and mobile preview
* If favicon still shows Replit:
  * Confirm favicon.ico returns 200 at https://arclo.pro/favicon.ico
  * Wait ~5–10 minutes (browser favicon cache is aggressive)

Acceptance criteria

* Browser tab shows Arclo logo (not Replit).
* Bookmark icon shows Arclo logo.
* Mobile home screen icon uses Arclo branding.
* No references to Replit favicon remain in page source.

This should be treated as a blocking branding bug, not cosmetic.
