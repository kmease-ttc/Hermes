## 1) Theme adjustments (lighter + bluer, still dark)

### A. Find the theme source of truth

* Locate where colors are defined:

  * `tailwind.config.*` (extend colors)
  * `globals.css` / `theme.css` / `index.css` (CSS variables like `--background`, `--card`, `--muted`, etc.)
  * any `ThemeProvider` / `tokens.ts` / `ui/theme.ts`
* Identify the current base background, surface/card, border, text, muted text, accent, and glass overlay colors.

### B. Make the darks less “editor-black”

Adjust the neutral ramp so the background is not near-black and surfaces are lifted.

* Background: lift 5–10% (perceptually) and tint slightly blue.
* Surfaces (cards/panels): lift a bit more than background so hierarchy is clear.
* Borders: slightly brighter than current, but low-contrast.
* Text: keep strong readability; soften pure-white to off-white.

Implementation guidance (not exact hex required, but keep consistent):

* Introduce a “cool neutral” ramp (blue-gray) rather than pure gray.
* Avoid high saturation; keep the tint subtle (barely noticeable).
* Keep accents minimal; use one primary accent (your existing gold) and one secondary (soft purple/blue), both desaturated.

### C. Reduce “code editor” feel

This usually comes from:

* Very dark background + very high-contrast bright text
* Strong neon accents
* Too much pure gray/black
  Fix by:
* Slightly lifting background/surfaces
* Using off-white for text instead of #fff
* Ensuring muted text isn’t too dim (improve readability)
* Avoiding “terminal green” / intense cyan hues

### D. Acceptance criteria for theme

* Main background is visibly not “near black”
* Cards/panels pop subtly without looking washed out
* App still reads as dark-mode
* No neon accents; only subtle blue tint in neutrals

## 2) Make the glass effect glossier / shinier (still tasteful)

Wherever you have glassmorphism (cards, modals, nav panels):

### A. Update glass base styles

* Increase blur slightly (but not foggy)
* Add a subtle top highlight gradient (specular sheen)
* Slightly increase border brightness and add a faint inner border
* Add a soft shadow that reads as depth, not heavy drop shadow

Implementation approach:

* Create a reusable class (e.g. `glass-panel`) or component style (e.g. `Card` variant)
* Use:

  * `background` with low-opacity surface color
  * `backdrop-filter: blur(...) saturate(...)`
  * `border` with slightly brighter alpha
  * `background-image: linear-gradient(...)` for highlight sheen
  * Optional: `box-shadow` with a soft, cool-toned shadow

### B. Avoid making it “wet plastic”

* Keep the highlight subtle
* Don’t crank saturation
* Don’t add strong reflections or bright streaks

### C. Acceptance criteria for glass

* Glass panels feel layered with a gentle sheen
* Borders look crisp but not bright
* No “milky” blur; content remains readable
